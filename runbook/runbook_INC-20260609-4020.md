# RUNBOOK TITLE:
Resolving `KeyError` Exception in `app.py` Script for "Failed Runs" Metric Display

# INCIDENT SUMMARY:
The `app.py` script is experiencing a `KeyError` exception when attempting to display the "Failed Runs" metric using `st.metric`. The error occurs because the `stats` dictionary does not contain the key `'errors'`. This incident is classified as medium severity, as it does not cause a complete system failure but prevents the display of an important metric.

- Familiarity with the `streamlit` framework and the `devops-autopilot` application
- Understanding of Python dictionaries and error handling

	* Command: `sed -i 's/stats\['errors'\]/stats.get('errors', 0)/g' app.py`
2. **Add try-except blocks**: Wrap dictionary key access in try-except blocks to catch and handle `KeyError` exceptions. For example:
	```python

    errors = stats['errors']
except KeyError:
    errors = 0
```
	* Command: `sed -i 's/stats\['errors'\]/try:\n    errors = stats\['errors'\]\nexcept KeyError:\n    errors = 0/g' app.py`

	* Command: `sed -i 's/stats\['errors'\]/if 'errors' in stats:\n    errors = stats\['errors'\]\nelse:\n    errors = 0/g' app.py`
